# CS330 - Computer Graphics and Visualization

### How do I approach designing software?

This project helped me improve my ability to break larger ideas into smaller pieces. When designing my scene, I started by identifying the main objects in the reference image and then determined which primitive shapes could be used to create them. This made the project easier to plan and implement. In future projects, I can use the same approach of breaking complex problems into smaller, manageable parts before starting development.

### How do I approach developing programs?

One development strategy I used throughout this project was building and testing incrementally. Instead of trying to create the entire scene at once, I added objects, textures, lighting, and camera controls one step at a time. Iteration played a major role in the project because I regularly adjusted object placement, lighting, and proportions based on testing and feedback. Throughout the milestones, I became more comfortable working with OpenGL and making changes to improve both the functionality and appearance of the scene.

### How can computer science help me in reaching my goals?

Computer Graphics and Visualization introduced me to concepts I haven't really touched before - 3D transformations, lighting models, textures, and camera systems. These skills expanded my understanding of how graphics applications and game engines work. As I continue my journey, this knowledge will no doubt help me better understand software development concepts related to graphics and user experiences. Professionally, the problem-solving and programming skills developed in this course can be applied to many areas of software engineering, even outside of graphics programming, by improving my ability to design, build, test, and refine complex software projects.
